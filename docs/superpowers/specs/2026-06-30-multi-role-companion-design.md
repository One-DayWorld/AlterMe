# AlterMe 多类型陪伴 — 设计文档

日期:2026-06-30
状态:已与用户确认设计,待评审后进入实现计划

## 1. 目标

把当前的「单一陪伴」升级为「多个可切换的陪伴角色」。典型用法:
- 👑 **女王** — 深夜
- 🌳 **树洞** — 想倾诉、被倾听时
- 🎒 **玩伴** — 周末旅游、轻松玩闹
- 🧠 **理性** — 白天工作,要冷静分析、给建议

每个角色有自己的人设、专属铁律、对话历史和羁绊;但**共享**同一份长期画像(因为都是同一个"你")。

## 2. 现状(改造起点)

当前为单陪伴架构:
- `persona`(一段自由文本人设)+ `sessionRules`(本场规则)
- 单条 `chatHistory`、单份 `profile`(facts/interests/commStyle/toneContract)、单套羁绊 `pet`
- `Brain.buildSystemPrompt(provider, persona, profileInject, rules)` 组装 system prompt
- 浏览器开发期 `storage.js` 自动回退 localStorage、`http.js` 回退 `fetch`

## 3. 核心决策(已确认)

| 维度 | 决策 |
|------|------|
| 角色之间记忆 | **长期画像共享**,各角色都认识同一个你 |
| 角色之间对话历史 | **各自独立**,女王不知道你和树洞聊了什么 |
| 羁绊等级 | **每个角色单独**升级 |
| 切换方式 | **手动切换为主**(顶部切换条) |
| 角色数量 | **可自由增删改** + 内置 4 个模板 |
| 迁移落点 | 现有数据直接迁入「女王」,**不建"默认"角色** |
| Mac 测试 | 浏览器(仅 UI),沿用 `npm run dev` |

## 4. 数据模型

### 4.1 角色对象

```
role = {
  id,       // 唯一标识,如 "queen" / "tree" / "buddy" / "rational" 或自动生成
  name,     // 显示名,如 "女王"
  emoji,    // 备用头像(无图片时显示),如 "👑"
  avatar,   // 头像图片(data URL,可空);有则优先于 emoji 显示
  persona,  // 性格人设(复用现有 persona 文本)
  rules,    // 该角色专属铁律(复用现有 sessionRules 概念,可空)
}
```

### 4.2 存储键(`Store`)

| 键 | 含义 | 共享/独立 |
|----|------|-----------|
| `roles` | 角色列表(JSON 数组) | — |
| `activeRoleId` | 当前激活角色 id | — |
| `chatHistory_<roleId>` | 各角色独立对话历史 | **独立** |
| `pet_<roleId>` | 各角色独立羁绊 {xp, level} | **独立** |
| `profile` | 长期画像 | **共享**(不变) |
| `provider` / `apiKey_*` | 模型与 Key | 共享(不变) |

> 旧的全局 `persona` / `sessionRules` / `chatHistory` / `pet` 在迁移后不再单独使用(见 §5)。

### 4.3 内置模板(首次启动预置 4 个)

| id | name | emoji | 人设要点 |
|----|------|-------|----------|
| queen | 女王 | 👑 | 高冷、居高临下、简短克制、偶尔毒舌但其实在意你;自称『本王』,称你『你』 |
| tree | 树洞 | 🌳 | 温柔、耐心倾听、不评判、多共情少说教;鼓励你倾诉 |
| buddy | 玩伴 | 🎒 | 活泼、轻松、爱玩梗、像出游搭子;主动出点子找乐子 |
| rational | 理性 | 🧠 | 冷静、有条理、就事论事、给可执行建议;克制情绪化表达 |

模板仅为初始文本,用户可随意编辑/删除。

## 5. 迁移(一次性,不丢老数据)

App 首次检测到尚无 `roles` 时执行:
1. 创建上述 4 个内置角色。
2. 若用户已有旧 `persona`/`sessionRules`,**写入「女王」角色**的 persona/rules(覆盖女王模板默认文本);若旧 persona 为空,女王沿用模板文本。
3. 旧 `chatHistory` → `chatHistory_queen`(迁入女王历史)。
4. 旧 `pet`(xp/level)→ `pet_queen`;其余三个角色羁绊从 Lv.1 起。
5. `activeRoleId` 设为 `queen`。
6. `profile` 原样保留(共享)。

迁移幂等:已存在 `roles` 则跳过。

## 6. System Prompt 改动(brain.js)

`buildSystemPrompt` 主体逻辑不变(仍吃 persona + rules)。**新增一段永久「人称纪律」块**,与「底层 AI 身份」「实事求是」同级,无论哪个角色都钉在 system prompt 里:

```
【人称纪律——必须严格遵守】
- "我"永远指你(当前这个 AI 角色);"你"永远指正在跟你说话的用户,绝不可说反。
- "我的"是属于你(AI)的;"你的"是属于用户的。
- 复述用户讲过的事,主语要用"你"(指用户),不要错说成"我"。
- 每次回答前快速自检:人称有没有指对人。
```

目的:修正模型在中文角色扮演里常把"你/我""你的/我的"说反的问题。

## 7. 对话逻辑改动(app.js)

`send()` 改动点:
- 读取**当前激活角色**的 persona / rules / `chatHistory_<roleId>`
- 历史存回**当前角色**的 key
- XP 入账给**当前角色**的 `pet_<roleId>`(聊天 +8、提炼 +15、喂文章 +25 均算给激活角色)
- `refreshBond()` 显示当前角色的等级 + 共享的 articleCount

画像注入、提炼缓冲(`Memory.runRefine`)、喂文章逻辑不变(画像共享)。

## 8. 界面改动

### 8.1 顶部角色切换条
聊天页顶部新增一排角色头像(图片或 emoji),当前角色高亮;末尾一个「＋」进入角色管理。点击切换:
- 保存当前屏 → 切 `activeRoleId` → 清屏并载入目标角色 `chatHistory_<roleId>`
- 刷新羁绊显示为目标角色

```
┌─────────────────────────────┐
│  👑   🌳   🎒   🧠   ＋        │ ← 切换条(当前高亮)
├─────────────────────────────┤
│  (女王的对话历史…)            │
│                             │
├─────────────────────────────┤
│ [说点什么…]          [发送]  │
└─────────────────────────────┘
```

### 8.2 设置 → 角色管理
把现有「性格人设 / 本场规则」两块,升级为**角色管理**:
- 角色列表:每项显示头像 + 名字,可编辑/删除(至少保留 1 个,防删空)
- 新建角色:名字、emoji、头像图片、人设、专属铁律
- 编辑沿用现有 persona / rules 文本框 UI

### 8.3 AI 头像(图片)
- 角色编辑里可上传头像图片;上传后用 `<canvas>` 缩放到约 128×128、导出为 JPEG data URL(`toDataURL('image/jpeg', 0.8)`)存入 `role.avatar`,控制体积(单张约 5–15KB,适配 Preferences)。
- 聊天里 **AI 气泡** 旁显示该角色头像(有图用图,无图用 emoji);用户气泡不加头像。

### 8.4 一键清屏(仅清对话框)
输入栏附近新增「清屏」按钮:
- 行为:`messages` 容器清空(可重新插入一条系统欢迎语)
- **不**改动 `chatHistory_<roleId>` —— 重开 App 或切走再切回会从历史恢复
- 非破坏性操作,单击即可,无需二次确认
- 与「设置 → 清空全部记忆」语义完全区分

## 9. 备份 / 恢复(storage.js)

`EXPORT_KEYS` 从静态列表改为**动态**:固定键(`profile` / `provider` / `apiKey_*` / `roles` / `activeRoleId`)+ 遍历 `roles` 拼出所有 `chatHistory_<roleId>` 和 `pet_<roleId>`。
导入时原样写回,保证换机后角色、历史、各角色羁绊、头像全部还原。
(注:头像图片随 `roles` 一并导出,会增大备份体积,属预期。)

## 10. Mac 测试环境

沿用现成 `npm run dev`(`cd www && python3 -m http.server 8080`),浏览器打开 `http://localhost:8080`:
- `storage.js` 自动回退 localStorage,角色/历史/羁绊/头像/清屏/设置均可测。
- **已知限制**:浏览器直连千问/DeepSeek 会被 **CORS 拦截**,AI 对话发不出去 → 浏览器仅用于 UI/交互测试;验证真实对话仍需真机(或后续如需可另上模拟器/代理,本期不做)。

## 11. 受影响文件

| 文件 | 改动 |
|------|------|
| `www/js/storage.js` | 角色 CRUD、activeRoleId、per-role history/pet、迁移、动态 EXPORT_KEYS |
| `www/js/brain.js` | 新增「人称纪律」永久块;`buildSystemPrompt` 其余不变 |
| `www/js/app.js` | 切换条渲染与切换、send 读写当前角色、头像渲染、清屏、角色管理 UI 接线 |
| `www/index.html` | 切换条容器、清屏按钮、角色管理区(替换原 persona/rules 区) |
| `www/css/style.css` | 切换条、头像、清屏按钮样式 |
| `www/js/memory.js` | 基本不变(画像共享);仅核对 XP 入账走当前角色 |

## 12. 范围之外(YAGNI,本期不做)

- 按时间/场景自动切换、时间段提示(留待后续)
- 浏览器端 AI mock / 本地 CORS 代理 / Android 模拟器
- 角色之间共享/转移对话历史
- 多头像/动图/语音

## 13. 验收要点

1. 首次升级后,旧人设与旧聊天出现在「女王」里,记忆面板内容不丢。
2. 顶部可在 4 个角色间切换,各自历史互不串台。
3. 每个角色羁绊独立升级、独立显示。
4. AI 气泡显示对应角色头像(图片优先,emoji 兜底)。
5. 清屏按钮只清当前屏,重开后历史还在。
6. 切到任一角色,问"你/我"相关问题,人称不再说反。
7. 导出再导入,角色、各自历史、各自羁绊、头像完整还原。
8. `npm run dev` 浏览器下 UI 全部可操作(对话因 CORS 不通,属已知)。
